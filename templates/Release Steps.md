# Release Steps

  

- Release meeting on Wednesday

- Release is tagged and priority identified in Jira

- Next release is created in Jira

- Prepare the new release page

- Release managers are selected and notified

- Release checklist wiki page is created

- Maintenance bridge is scheduled

- Change request is created

- Change request is approved

- Change request is represented in the CAB meeting (approvals are provided by noon the day before)

- Production release happens during the maintenance bridge

- Tickets are moved to done

- Release versions are added to the release description in Jira

- Release is marked as done

- Quick filters are updated with the next release

- Release notes are sent to the SLEM distribution list

- Archive any epic channels that are no longer needed.

- Update the status and plan end date of epic tickets included in the release

- Change Request is closed
